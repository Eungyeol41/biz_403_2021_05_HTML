Flex를 이요한 화면구현

초창기 HTML은 위에서 아래 방향으로 진행되는 Layout이 주로 사용되었다.
가로방향으로 Content를 배열하는 Layout이 필요
여러가지 Hack(Tip)들이 등장을 하고 다양한 방법이 동원된다.
표준 HTML 사용법과는 다소 거리가 있어서 여러가지 문제들을 일으키고 경우에 따라서는 원하는대로 Layout이 만들어지지 않았다.
대표적 => div box를 floating하여 배열하는 방법

우리나라만세

box model tag를 float로 설정을 하면 box width에 따라 화면에서 한 라인에 여러 tag를 배열할 수 있다.
그런데 float로 만든 Layout은 floating되기를 원하지 않는 tag에 까지 영향을 미치게 된다.

inline-block

float를 사용하여 Layout을 구성하던 HTML 디자이너들에게 반가운 소식이 전해진다.
HTML5의 표준에 box model을 변경할 수 있는 속성이 추가된다.

display 속성

기본적인 box model을 변형(변경)하여 적용할 수 있다.
block -> inline
inline -> block

div tag에 display를 inline으로 설정했더니 갑자기 block 성질을 버리고 한 라인에 여러 개의 box를 배열할 수 있게 되었다.
그런데 inline 속성은 box의 width를 설정할 수 없다.
다행히 inline 속에서 block의 성질을 포함한 inline-block 속성이 추가된다.
드디어 div box를 사용하여 원하는대로 width를 설정하고 한 라인에 여러 박스를 배치할 수 있게 되었다.
